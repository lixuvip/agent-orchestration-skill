# v0.1.3 Release Notes

`v0.1.3` adds Project Autopilot guidance for recurring Codex automation that keeps a project moving toward explicit done criteria.

## What Changed

- Added `PROJECT_AUTOPILOT.md` for goal-driven recurring automation across repositories, branches, issues, PRs, tests, and release checklists.
- Added English and Chinese templates for project goal contracts, automation plans, automation ticks, automation memory, escalation reports, and `AGENTS.md` guidance snippets.
- Added a repository-level `AGENTS.md` so future maintenance runs inherit validation, install-sync, documentation, and release boundaries.
- Updated the main skill trigger description with continuous-progress phrases such as project autopilot, recurring automations, `持续推进`, `一直做到目标效果`, and `定时自动继续`.
- Expanded controller-loop, intake, monitoring, and workflow references to distinguish heartbeat follow-up from cron-style workspace autopilot.
- Added a copy-ready continuous project autopilot example.
- Strengthened validation and smoke tests so the new autopilot resources remain discoverable and structurally complete.

## Why It Matters

- **Project rules persist**: recurring automation can read `AGENTS.md` / `AGENTS.override.md` instead of rediscovering repository expectations every run.
- **Automation has a target**: the goal contract defines done criteria, allowed autonomous actions, verification commands, cadence, memory path, and stop conditions.
- **Runs stay idempotent**: automation memory tracks the latest effective update, posted messages, blockers, and next safe action.
- **Progress stays bounded**: each tick takes one safe action and escalates before merge, push, deploy, destructive changes, public API contract changes, or scope expansion.
- **Long-running work becomes recoverable**: cron automations can continue workspace progress while heartbeat automations handle current-thread callbacks.

## AGENTS.md And Codex Automation

For target projects, put durable project rules in `AGENTS.md` or nested `AGENTS.override.md`. Put temporary goal state in automation memory, not in `AGENTS.md`.

Use `.codex/config.toml` for Codex configuration and fallback instruction-file settings. Do not store secrets, live task state, or project memory there.
