# v0.1.2 Release Notes

`v0.1.2` turns `agent-orchestration` from a role-dispatch template into a tighter Codex coordinator workflow for real branch, thread, and handoff work.

## What Changed

- Added orchestration intake rules for deciding when to ask before creating branches, worktrees, user-visible threads, callbacks, heartbeat automations, merges, pushes, or PRs.
- Added a controller loop for main-thread coordination across child threads, subagents, branches, worktrees, status requests, callbacks, and merge-readiness checks.
- Added English and Chinese templates for orchestration intake, coordinator callbacks, status requests, and merge readiness.
- Expanded task dispatch and role reply templates with branch/worktree, commit, callback, and merge-policy fields.
- Upgraded heartbeat monitoring so it can request missing status or verification before summarizing terminal states.
- Added a branch callback controller-loop example for engineering branch work, read-only QA, and coordinator-controlled merge readiness.
- Strengthened validation and smoke tests so these new resources remain discoverable and structurally complete.

## Why It Matters

- **Less repeated clarification**: Codex gets a standard intake gate for when to ask and when to proceed.
- **Cleaner main-thread control**: child threads can report back with branch, commit, verification, risks, and recommended coordinator action.
- **Safer branch finalization**: merge and push actions now have a checklist instead of relying on informal summaries.
- **Better long-running work**: heartbeat monitoring can detect silence, missing verification, and non-terminal states without guessing.
- **More reusable team workflow**: the same pattern works for engineering, QA, review, release prep, and multi-repository finalization.
