# v0.2.0 — Progressive Orchestration

`v0.2.0` turns `agent-orchestration` into a progressive coordination system: Lite work stays in the current conversation, Standard work loads one coordination pack, and Durable work adds one Autopilot pack for recurring progress and recovery.

This release combines the reliability work developed after `v0.1.4`—versioned callbacks, commit-pinned gates, fenced automation, lifecycle cleanup, bounded external review/research, auditable installation, and scale budgets—into one smaller operating model.

## Highlights

### Less context, clearer routing

- Reduced `SKILL.md` from 98 to 42 lines.
- Consolidated 14 overlapping core references into four bilingual capability-pack files.
- Added deterministic `Lite`, `Standard`, and `Durable` selection through `scripts/route_orchestration.py`.
- Made the router return the exact reference and template set required by the task.
- Added `scripts/scale_test.py` to prevent entrypoint and capability-pack growth from silently regressing.

| Mode | Intended use | Loads |
| --- | --- | --- |
| Lite | One-shot current-context work | No core pack |
| Standard | Multi-role, async, cross-repo, finite monitoring, or formal gates | One language coordination pack plus selected templates |
| Durable | Recurring work or recovery across ticks | Standard pack plus one language Autopilot pack |

### Reliable callbacks and acceptance

- Added `ORCHESTRATION_EVENT_V1` with goal/task ID, attempt, dispatch nonce, coordinator epoch, unique event ID, timestamp, role/thread identity, and exact artifact identity.
- Duplicate events and stale attempts, nonces, epochs, or SHAs are successful no-ops.
- Role execution status, QA/review gate verdict, and coordinator state are separate dimensions.
- Role `DONE` enters `IN_REVIEW`; delivery requires coordinator `ACCEPTED` and current evidence.
- QA/review verdicts are pinned to the exact inspected SHA and are invalidated by later code commits.

### Recoverable automation

- Added file-locked leases with expiry, renewal, takeover protection, and monotonic fencing tokens.
- Every side-effecting tick verifies ownership before messages, external writes, cleanup, or memory commits.
- Added monotonic heartbeat shutdown: `ACTIVE -> DRAINING -> CLOSED`.
- Final summaries and cleanup requests are idempotent; late callbacks cannot recreate a closed monitor.
- Durable ticks compare the latest effective update and action/message/event keys so unchanged runs stay quiet.

### Project Autopilot

- Added goal-contract, automation-plan, tick, memory, escalation, and stable-project-guidance templates.
- Separated durable `AGENTS.md` rules from transient automation memory.
- Limited every tick to one smallest authorized safe action.
- Kept merge, push, deploy, publish, destructive changes, secrets, spending, public API changes, and product-scope expansion behind explicit authority.

### Bounded `agy` / Gemini review and research

- Added separate external review and external research workflows.
- Gemini always runs through local `agy`, never the standalone `gemini` CLI.
- Added sandboxed print execution, host timeout, output guards, and allowlisted context bundles.
- Added independent Codex/Gemini comparison reports and structured quality evaluation.
- Quality logs default to the Codex-owned external-review ledger outside the target repository.
- Target `AGENTS.md` changes, project-local logs, whole-repository disclosure, and any external edits remain separately authorized writes.

### Auditable installation and release checks

- The installer now rejects dirty sources by default, stages replacements, records source commit and tree hash, retains the previous install, and verifies installed parity.
- Added `--dry-run`, explicit `--allow-dirty`, and `--restore` flows.
- CI and installation run static validation, smoke tests, forward tests, protocol behavior, automation concurrency/lifecycle, routing behavior, scale budgets, built-in skill validation, and `git diff --check`.

## Migration From v0.1.x

The runtime behavior is compatible at the workflow level, but prompts or tools that reference old internal Markdown paths must be updated.

| Removed reference family | Replacement |
| --- | --- |
| Intake, routing, communication, controller, protocol, state, workflow, monitoring, and concurrency references | `COORDINATION_RUNBOOK.md` or `COORDINATION_RUNBOOK.zh-CN.md` |
| Project-instruction discovery and automation-tooling references | `PROJECT_AUTOPILOT.md` or `PROJECT_AUTOPILOT.zh-CN.md` |
| Repeated callback identity fields in human role replies | `coordinator_callback.template.md` machine envelope |

Do not load both language versions. Use the capability map at `skills/agent-orchestration/references/README.md` when selecting templates.

## Upgrade

```bash
git pull
./scripts/install.sh
```

The installer validates the source, retains the previous local copy, installs the new skill, records provenance, and verifies parity. Roll back the retained copy with:

```bash
./scripts/install.sh --restore
```

## 中文说明

`v0.2.0` 把 `agent-orchestration` 升级为渐进式编排系统：Lite 留在当前对话，Standard 只加载一个协调能力包，Durable 再增加一个 Autopilot 能力包，用于周期推进和跨 tick 恢复。

这一版本把 `v0.1.4` 之后增加的版本化回调、commit 固定门禁、fenced automation、生命周期清理、有界外部审查/调研、可审计安装和规模预算统一成一条更轻的运行路径。

### 主要变化

- `SKILL.md` 从 98 行压缩到 42 行。
- 14 份重叠核心参考合并为 4 个中英文能力包文件。
- 路由器会输出任务实际需要的参考文件和模板，不再按模式整套预加载。
- `ORCHESTRATION_EVENT_V1` 可以拒绝重复和过期回调。
- QA/Review 绑定精确 SHA；后续代码 commit 会让旧证据失效。
- recurring tick 使用文件锁 lease 和单调 fencing token 防止重叠副作用。
- heartbeat 按 `ACTIVE -> DRAINING -> CLOSED` 单向收口。
- Project Autopilot 每 tick 只执行一个已授权安全动作，无变化时保持安静。
- 外部审查和外部调研分开处理；Gemini 只通过本机 `agy` 运行。
- 安装器记录来源 commit、tree hash 和 dirty 状态，保留上一版并验证安装一致性。
- 新增 scale budget 测试，防止入口和能力包重新膨胀。

### 从 v0.1.x 迁移

工作流语义保持兼容，但如果旧 prompt 直接引用了已删除的内部 Markdown 文件，需要改成：

- 协调、路由、协议、状态、巡检和并发相关内容：`COORDINATION_RUNBOOK.zh-CN.md`
- 项目指令、cron、memory 和 Autopilot：`PROJECT_AUTOPILOT.zh-CN.md`
- 异步机器回调：`coordinator_callback.zh-CN.template.md`

中英文能力包只加载一个版本。模板选择可查看 `skills/agent-orchestration/references/README.md`。

升级命令：

```bash
git pull
./scripts/install.sh
```

如需恢复安装器保留的上一版本：

```bash
./scripts/install.sh --restore
```
