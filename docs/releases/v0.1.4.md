# v0.1.4 Release Notes

`v0.1.4` hardens the Project Autopilot workflow by making its forward-test expectations visible, documented, and CI-checked.

## What Changed

- Added `scripts/forward_test.py`, a deterministic guard that checks the forward-test scenarios and core Autopilot references for heartbeat/cron routing, automation memory, latest effective update handling, no-op GitHub polling, and escalation gates.
- Added the forward-test guard to GitHub Actions and repository maintenance instructions.
- Added Project Autopilot loop diagrams for English and Chinese READMEs.
- Updated README, Chinese README, examples, forward-test docs, and changelog so release users can see what the new guard validates and how to run it locally.

## Why It Matters

- **Trigger quality is testable**: expected behavior for heartbeat callbacks, cron Autopilot, GitHub no-op polling, and missing `AGENTS.md` guidance is now checked before release.
- **Autopilot is easier to explain**: the loop diagram shows how goal contracts, project rules, automation ticks, safe actions, verification, and memory updates fit together.
- **Maintenance gets safer**: CI now catches accidental removal of the exact cues that make the skill choose the right workflow and avoid duplicate comments.

## How To Use It

Run the full local validation set before publishing skill changes:

```bash
python3 scripts/validate.py
python3 scripts/smoke_test.py
python3 scripts/forward_test.py
python3 ~/.codex/skills/.system/skill-creator/scripts/quick_validate.py skills/agent-orchestration
```
