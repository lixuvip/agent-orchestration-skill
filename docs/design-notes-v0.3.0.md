# v0.3.0 Design Provenance

This release combines a native-first simplification of the existing skill with selected reliability patterns inspired by the open-source [xAI Grok Build repository](https://github.com/xai-org/grok-build). The fixed inspection baseline was repository commit [`ba76b0a683fa52e4e60685017b85905451be17bc`](https://github.com/xai-org/grok-build/tree/ba76b0a683fa52e4e60685017b85905451be17bc), inspected on 2026-07-20. That snapshot declares source revision `ba69d70c2f7d70a130a323b2becdf137af784c7f`.

Grok Build is distributed under the [Apache License 2.0](https://github.com/xai-org/grok-build/blob/ba76b0a683fa52e4e60685017b85905451be17bc/LICENSE). This repository uses the sources below as design references. It does not copy Grok Build runtime code, tool implementations, event loops, or verifier prompts into `agent-orchestration`.

## What remains native to this repository

The six-file runtime, current-task-first routing, distinction between internal subagents and user-owned sidebar tasks, Local/Worktree handoff semantics, verification budget, separate `agy-second-opinion` skill, quiet automation behavior, and installer parity checks are local design decisions grounded in Codex's available product surfaces and this repository's previous field use.

## Explicit non-adoptions

- No Grok-specific agent types, personas, ACP methods, session files, event streams, stop-gate code, scheduler, hooks, or permission configuration are introduced.
- No verifier prompt is copied. Requirement closure is a coordinator responsibility unless risk or the user justifies independent review.
- No mandatory three-pass verification loop is introduced.
- No Best-of-N candidate is selected merely because it is the least bad; rejecting all candidates is valid.
- No role registry, callback envelope, task board, heartbeat file, lease, or deterministic routing script returns to the installed orchestration skill.
